# 管理员账号生命周期

## 结论

管理员账号采用不可物理删除策略。产品中的“移除管理员访问权”统一通过停用完成，不提供 `DELETE` 接口。

## 生命周期

1. 创建：超级管理员创建账号并分配角色。
2. 启用：账号可以按当前角色登录管理控制台。
3. 停用：立即撤销现有会话并禁止继续登录，等同产品层面的删除语义。
4. 恢复：在管理员列表选择“已停用”状态，重新启用账号。

## 为什么不物理删除

- 保留管理操作的审计主体。
- 保留预测、策略、回测和发布治理的责任追踪。
- 避免历史记录因外键或匿名化失去解释能力。
- 支持误停用后的受控恢复。

## 验收

- 管理员页面不存在“删除账号”按钮。
- API 不提供管理员 `DELETE` 路由。
- 停用后账号无法登录，现有会话失效。
- 停用账号仍可通过状态筛选查看。
- 重新启用后可按原有或调整后的角色登录。
- 停用与恢复均写入审计记录。
