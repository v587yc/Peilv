# 足球赔率分析系统修复任务表

> 依据系统审计结果整理。建议按 P0 → P1 → P2 顺序执行。
>
> 状态说明：`未开始` / `进行中` / `待验证` / `已完成` / `阻塞`

## 一、总体目标

将当前“前端驱动的半自动实验系统”修复为具备以下能力的可靠系统：

1. 数据库结构、代码模型和初始化迁移保持一致；
2. 赔率、预测、验证和报表数据可追溯、可重复、不可错误覆盖；
3. 缺失数据不会被误判为正确；
4. 验证、报表和学习使用统一口径；
5. 学习结果真正影响后续规则和预测；
6. 自动任务不依赖浏览器页面存活；
7. 回测数据与线上数据完全隔离；
8. API 具备明确的认证、授权和输入边界；
9. 具备自动化测试、日志、重试和失败告警能力。
